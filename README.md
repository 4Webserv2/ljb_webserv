## Best practices

- **SOLID**: software design principles  
- **DRY**: Don't Repeat Yourself
- **CONVENTIONAL COMMITS**: conventions for commit messages  
    - **feat**: new feature  
    - **fix**: bug fix  
    - **docs**: documentation  
    - **style**: formatting  
    - **refactor**: code refactoring  
    - **perf**: performance improvement  
    - **test**: adding tests  
    - **chore**: maintenance  
    - **build**: project build
